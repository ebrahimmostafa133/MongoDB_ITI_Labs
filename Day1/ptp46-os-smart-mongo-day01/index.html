MongoDB Cheatsheet

Quick reference for MongoDB commands, operations, and query syntax. Essential commands for database operations, CRUD, aggregation, and more.

Database Operations

Show Databases
Query
show dbs

Lists all databases on the MongoDB server

Example:
show dbs
Use Database
Command
use <database_name>

Switches to a specific database (creates it if it doesn't exist)

Example:
use myDatabase
Current Database
Query
db

Displays the current database name

Example:
db

Collection Operations

Show Collections
Query
show collections

Lists all collections in the current database

Example:
show collections
Create Collection
Command
db.createCollection("collection_name")

Creates a new collection with the specified name

Example:
db.createCollection("users")
Drop Collection
Delete
db.<collection_name>.drop()

Removes a collection from the database

Example:
db.users.drop()

CRUD Operations

Insert Document
Create
db.<collection>.insertOne({field: value})

Inserts a single document into a collection

Example:
db.users.insertOne({name: "John", age: 30, city: "NYC"})
Find Documents
Query
db.<collection>.find({query})

Retrieves documents that match the query criteria

Example:
db.users.find({age: {$gt: 25}})
Update Document
Update
db.<collection>.updateOne({filter}, {$set: {update}})

Updates a single document matching the filter

Example:
db.users.updateOne({name: "John"}, {$set: {age: 31}})
Delete Document
Delete
db.<collection>.deleteOne({filter})

Deletes a single document matching the filter

Example:
db.users.deleteOne({name: "John"})

Query Operators

Comparison Operators
Query
$eq, $ne, $gt, $gte, $lt, $lte, $in, $nin

Compare values in queries

Examples:
db.users.find({age: {$gt: 25}})
db.users.find({status: {$in: ["active", "pending"]}})
Logical Operators
Query
$and, $or, $not, $nor

Combine multiple query conditions

Example:
db.users.find({$or: [{age: {$lt: 25}}, {status: "active"}]})
Element Operators
Query
$exists, $type

Check for field existence or data type

Example:
db.users.find({email: {$exists: true}})

Aggregation Framework

Aggregation Pipeline
Query
db.<collection>.aggregate([{$stage1}, {$stage2}])

Process data through multiple pipeline stages

Example:
db.orders.aggregate([ {$match: {status: "completed"}}, {$group: {_id: "$product", total: {$sum: "$amount"}}} ])
Common Stages
Query
$match, $group, $sort, $project, $limit, $skip

Frequently used aggregation pipeline stages

Example:
db.users.aggregate([ {$match: {age: {$gt: 20}}}, {$sort: {age: -1}}, {$limit: 10} ])
Aggregation Operators
Query
$sum, $avg, $min, $max, $push, $addToSet

Operators used within $group stage

Example:
db.sales.aggregate([ {$group: {_id: "$region", total: {$sum: "$amount"}, avg: {$avg: "$amount"}}} ])

Index Operations

Create Index
Command
db.<collection>.createIndex({field: 1})

Creates an index on a field (1 for ascending, -1 for descending)

Example:
db.users.createIndex({email: 1})
List Indexes
Query
db.<collection>.getIndexes()

Displays all indexes on a collection

Example:
db.users.getIndexes()
Drop Index
Delete
db.<collection>.dropIndex("index_name")

Removes a specific index from a collection

Example:
db.users.dropIndex("email_1")
